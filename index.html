¿Qué es Fut 7 El Jaguar?

Sistema Integral para Torneos de Fútbol 7

Fut 7 El Jaguar es una aplicación web full-stack para organizar y gestionar torneos de fútbol 7 con múltiples categorías. Cuenta con un backend REST en Node.js/Express conectado a PostgreSQL via Docker, y un frontend puramente en HTML, CSS y JavaScript Vanilla con autenticación por sesión, soporte de modo oscuro/claro, y múltiples módulos especializados.

🐆
Login Seguro
Pantalla de acceso con diseño de cancha aérea animada. Autenticación via sessionStorage con protección de rutas en cada módulo.
🏆
Multi-Categoría
Todo el sistema es por categoría. Puedes tener Juvenil, Mayor, Femenil, etc. con sus propios equipos, jugadores, rol y estadísticas independientes.
👥
Equipos y Jugadores
Registro de equipos con color, escudo y datos. Gestión completa de plantilla con foto, número de jersey y estado activo/baja.
📅
Rol de Juegos
Generación automática del calendario de partidos tipo round-robin. Jornadas doradas, partidos extra, marcado de jornada actual y descarga PDF.
⏰
Horarios Drag & Drop
Agenda semanal interactiva. Arrastra partidos desde el rol hacia celdas de día/hora. Soporte para mover partidos ya agendados a otro slot.
🏆
Puntos & Goles
Tabla de posiciones automática con PJ, G, E, P, GF, GC, DG y Pts. Registro de goles y penales por jugador. Rankings individuales.
🎨
Editor de Diseños
Editor visual de plantillas tipo canva. Crea diseños con texto, formas, gradientes, colores de equipo y logos. Asigna diseños por categoría.
📊
Gráficas
Dashboard de estadísticas con Chart.js. Partidos por día, rendimiento mensual, configuración de costos de credencial y premios por posición.

Funcionalidades Clave

Todo lo que Incluye

✓Sistema Multi-Categoría: Cada categoría (Juvenil, Mayor, Femenil, etc.) tiene su propio conjunto de equipos, jugadores, rol, horarios, goles y estadísticas completamente independientes.
✓Generador Round-Robin Automático: Con solo presionar "Crear Rol" genera el calendario completo. Soporte de número par e impar de equipos, partidos en casa y fuera balanceados.
✓Drag & Drop de Horarios: Sistema nativo de arrastre de partidos desde el panel de pendientes hacia la cuadrícula semanal. También permite mover partidos ya agendados a otro slot.
✓Jornadas Doradas: Marca jornadas completas como jugadas para generar estadísticas y desbloquear la vista de sugerencias de próximos partidos.
✓Exportación a PDF: El módulo de Rol de Juegos permite exportar el calendario completo de partidos como PDF directamente desde el navegador.
✓Colores Dinámicos por Equipo: En la vista de jugadores, todos los colores del UI (encabezado, acentos, botones) se adaptan automáticamente al color registrado del equipo.
✓Editor Visual de Plantillas: Herramienta tipo canvas con capas de texto, formas, gradientes y más de 20 tipografías para crear diseños de equipos asignables por categoría.
✓Tabla de Posiciones Automática: Calcula PJ, G, E, P, GF, GC, diferencia de goles y puntos. Top 3 resaltados en oro, plata y bronce.
✓Modo Oscuro / Claro: Toggle persistente en todos los módulos. Preferencia guardada en localStorage con transiciones suaves de tema en toda la UI.
✓Imágenes Base64 Almacenadas: Escudos de equipos y fotos de jugadores se convierten a base64 y se guardan en la tabla publico de PostgreSQL sin necesidad de CDN externo.
✓Sugerencia de Partidos: Módulo inteligente que, analizando las jornadas ya jugadas, sugiere los partidos más lógicos para programar a continuación.
✓Gestión de Goles y Penales: Registro individual de goles y penales por jugador, con ranking de goleadores y tabla de jugadores más amonestados.
✓Configuración de Ingresos: Define costo de credencial por jugador y montos de premios para los 4 primeros lugares del torneo. Visible en el dashboard de gráficas.
✓Docker Compose Listo: La base de datos PostgreSQL levanta en un contenedor con el esquema ya aplicado. Inicio con un solo comando ./start.sh.
✓Protección de Rutas: Cada módulo verifica sessionStorage.getItem('sesion_jaguar') al cargarse y redirige al login si no hay sesión activa.
✓Splash Screen Inteligente: Animación de carga con el logo del jaguar solo aparece en la primera visita al menú por sesión. En navegaciones posteriores se omite para mayor fluidez.

Las tablas rol, goles, penales, horarios y colores usan JSONB para almacenar estructuras de datos complejas como arrays de jornadas, mapas de partidos y configuraciones completas. Esto permite actualizar el estado completo del torneo en un solo request con ON CONFLICT ... DO UPDATE (upsert). El campo asignado_a en disenos es un array de texto de PostgreSQL para filtrar diseños por categoría.

Instalación y Arranque

Cómo Correr el Proyecto

El proyecto está diseñado para arrancar fácilmente con Docker y el script de inicio incluido. Solo necesitas Docker Desktop y Node.js instalados.

Requisitos: Docker Desktop (para PostgreSQL), Node.js v18+, y un navegador moderno. El frontend es HTML puro — se abre directamente como archivo local.

# 1. Clonar o descomprimir el proyecto
cd team-soccer-fixed

# 2. Opción A — Script automático (recomendado)
chmod +x start.sh
./start.sh
# Levanta Docker, espera a Postgres, instala dependencias y arranca el backend

# 2. Opción B — Manual paso a paso
docker compose up -d              # Levanta PostgreSQL en puerto 5433
cd backend
npm install
node server.js                    # API en http://localhost:3001

# 3. Abrir el frontend en el navegador
# Abre este archivo directamente:
file:///ruta/al/proyecto/frontend/login/index.html

# Credenciales de acceso:
Usuario:    jaguar
Contraseña: jaguar123
# Variables de entorno del backend (opcionales, con defaults):
PORT         = 3001        # Puerto del servidor Express
DB_HOST      = localhost   # Host de PostgreSQL
DB_PORT      = 5433        # Puerto mapeado por Docker
DB_NAME      = teamsoccer  # Nombre de la base de datos
DB_USER      = postgres
DB_PASSWORD  = postgres
ℹEl script start.sh incluye un loop until pg_isready que espera a que Postgres esté completamente listo antes de arrancar el backend, evitando errores de conexión.
ℹEl esquema SQL se aplica automáticamente al primer arranque del contenedor via docker-entrypoint-initdb.d. No necesitas correr migraciones manualmente.
ℹLos datos persisten en el volumen Docker teamsoccer_data. Para borrar todo y empezar limpio: docker compose down -v.
ℹEl frontend funciona como archivos estáticos (file://), no necesita servidor web propio. Solo requiere que el backend esté corriendo en localhost:3001.
